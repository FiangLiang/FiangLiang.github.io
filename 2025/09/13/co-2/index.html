1 Introduction

关于 Instruction set architectures ( ISA, 指令集架构 )，我在第一章提过，但当时讲的不是很清楚，我看到NoughtQ的笔记本 上讲的很详细，所以再在下面提一下。

介绍一下这一节提到的几个概念

Instruction set architectures：一种抽象，定义了计算机处理器可以执行的所有指令，以及如何处理这种指令。完整的 ISA 包含指令的种类、执行方式以及操作数。

Instruction set：指令集，可分为精简指令集计机（reduced instruction set computer, RISC）和复杂指令集计算机（complex instruction set computer, CISC）

store-program concept：指令和各种数据都以数字的形式被储存

还可以顺便学几个生词：

time-honored：历史悠久的

rationale：原理

palatable：可口的

3 Operands of the Computer Hardware

有点杂，姑且分为两部分

Register

Register（寄存器）是Processor 内用来暂存指令、数据和地址的内部存储器。

  • 在RISC-V 架构中，共有32个 Registers，每个 Register 的 size 为64 bits，分别命名为 x0~x31，其中x0被保留用来存放常数 0
  • 64 bits 的组合称为 doubleword，32 bits 的组合称为 word
  • Arithmetic instructions 中设计的三个 operands 必须来自 Registers，而不是 Memory
  • Register 的数量之所以只有32个，是因为太多Register会提高 clock cycle，从而降低效率。由此可以引出第二个 Design Principle：Smaller is faster
  • Register 在能耗和效率方面都远胜于 Memory，因此二者中间产生了 cache 以提高效率

Data transfer

由于Regster数量有限，当遇到如数组、结构体等数据量较大的数据结构时，只能将其存放在 Memory 中。而内存中的数据无法直接进行 arithmetic operations，需要将其 transfer 到 Registers 中

  • 调用 Memory 中的数据需要提供 Address，Address类似数组的索引
  • data transfer instruction 分为 load 和 store
    • load指令中，最常用的是ld，全称为load doubleword，用以加载双字数据
    • load格式：load reg, offset(mem_base_addr)，等价于C语言中reg=mem_base_addr[offset/width]
    • store指令中，最常用的是sd，全称为store doubleword，用以存储双字数据
    • store使用格式与 load 基本一致
  • RISC-V 使用地址时采用 little-endian（小端序），数据低位在低位地址

4 Signed and Unsigned Numbers

主要在讲计算机中数的表示，相似内容在之前的课中提到不止一次了，这里就简单列一些点，权当复习

  • 虽然我们平时用十进制，但一个数其实可以在任意进制下表示
  • 计算机普遍使用二进制，用 least significant bit 代指 the rightmost bit，用 most significant bit 代指 the leftmost bit
  • 如果运算结果不能被正确表示，我们称发生了 overflow
  • 计算机中有符号数一般用 two’s complement representation，即补码储存

  • ld，lh，lb等涉及符号数的load操作，会采用 sign extension，复制符号位来填充64位的寄存器
  • 反码又称 one’s complement

5 Representing Instructions

RISC-V 的所有指令长度均为 32 位，每条指令由一块块字段组成，每块字段由不同的功能

常见的字段有：

  • opcode ：操作码，用于区分指令格式
  • funct3：额外的 opcode 字段，用于区分属于同一指令格式的不同指令
  • funct7：额外的 opcode 字段，用于区分属于同一指令格式的不同指令
  • rd：寄存器目标操作数，用于保存计算结果
  • rs1：第一个寄存器源操作数
  • rs12：第二个寄存器源操作数
  • immediate：立即数，即常数，以二进制补码的形式被存储，不能超过 32 位

指令格式一共有六种：

  • R-type：包含各种算术逻辑运算，格式：inst rd,rs1,rs2

  • I-type：包含有常数参与的算术逻辑运算、加载操作、jalr 指令，格式：inst rd,rs1,imm 或 inst rd,imm(rs1)

  • S-type：存储操作，格式：inst rs2,imm(rs1)

  • SB-type：条件分支，格式：inst rs1,rs2,imm[12:1]，这里为了扩大范围，默认 imm[0] 为 0

  • UJ-type：无条件分支，格式：inst rd,imm[20:1]，默认 imm[0] 为 0，理由相同；无条件分支可用于提高条件分支的跳转距离

  • U-type：高位立即数相关操作，格式：inst rd,imm[31:12]，可以配合加法将32位常数赋值到寄存器中

8 Supporting Procedures

Procedure

Procedure，直译为“过程”，相当于 C 语言中的函数，可以传入参数和重用

执行 Procedure 的步骤：

  1. 将参数放在过程可以访问得到的寄存器中
  2. 通过 jar 指令将控制权转交给指令
  3. 执行目标任务
  4. 将结果放在过程可以访问得到的寄存器中
  5. 通过 jalr 指令将控制权转交给指令

调用 Procedure 访问的寄存器：

  • x10-x17：参数寄存器，负责存储传递参数或返回值，同一个寄存器可能会同时作为参数和返回值。如果参数超过 8 个就需要放在栈上，栈指针是 x2，也叫 sp 始终指向栈顶元素
  • x1：用来保存返回地址，也叫做 ra，故其实 jalr x0 ,0(x1) 就相当于return 0

Stack

  • stack，栈，本质上是一块内存。因为寄存器空间有限，我们有时需要将不常用的（或之后再用的）变量从寄存器移到内存，用于储存这些“溢出”的变量的数据结构就是栈。
  • 栈指针之前提过，用 x2 ，也就是 sp 表示
  • 栈常用操作为 push 和 pop
    • push：如果要压入 nnn 个数据，先 addi sp,sp,-8*n，再逐一 sd 各个寄存器的值
    • pop：如果要弹出 nnn 个数据，先逐一 ld 各个寄存器的值，再 addi sp,sp,8*n
