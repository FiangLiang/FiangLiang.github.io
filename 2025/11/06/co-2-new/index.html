Instructions: Language of the Computer

重修版本，主要参考教材，内容会更加详细。

图表以及部分代码来源于前辈的笔记，包括：咸鱼暄的代码空间，NoughtQ的笔记本。伟大无需多言。

2.1 Introduction

这一章主要学习机器语言。

虽然说是机器语言，我们肯定不会让你去记那些 32 位的二进制指令。我们采用单词的形式去表示不同功能的二进制指令，这些单词，或者说单词表示的这些指令的集合，就叫指令集。

指令集是人规定的，没有统一的版本，不过大都彼此相似，我们主要学的是 RISC-V。

RISC-V 是一个基于精简指令集（RISC）原则的开源免费指令集架构，具有开放、灵活、可拓展的特点，被誉为芯片界的 “Linux”。嗯，不过现在的个人电脑或者服务器大多用的还是 Intel x86 架构，其特点是性能强大、生态成熟的闭源商业复杂指令集。

我猜大概是因为 RISC-V 比较简单，而且是开放的，这门课才会以 RISC-V 为主。（话说 Intel x86 是普通人能学到的吗？我不造啊）

不得不感慨一句：开源精神还是太伟大了。

这一节我们讨论一下操作参数

在 RISC-V 架构中，我们用 x0 - x31 分别表示32个64位寄存器，它们作为参数来支持指令的执行。其实，除了常数之外，指令中的操作参数只能是寄存器，不能是是内存中的数。事实上，如果你想要调用内存中的数据，必须要先使用 load 指令把内存中的数转移到寄存器中，然后才能参与运算。

那你会问了，既然寄存器这么有用，为什么规定只有32个呢？其一，32是一个很好的数字，它是二的幂次方；其二，后面你会发现，32个寄存器各有千秋，已经能基本满足需求；其三，大规模会一定程度上降低指令运行的性能。

Data transfer

当然，只靠 32 个寄存器肯定是没法完成大多数需求的。如果我们需要数组什么的来辅助操作，就必须依赖内存。

我们先来了解一下内存结构，内存在处理器的外面。内存地址从零开始，以字节为单位。比如我们往内存里塞两个 doubleword ，我们知道一个 doubleword 占8字节。所以第一个 doubleword 的地址是 0，第二个 doubleword 位置就是 8。不过上面指的是小端序，如果是大端序，第一个 doubleword 地址就是 7，第二个 doubleword 就是 15。也就是说大端序小端序存是一样存，只是地址的表达有所不同。我们学的 RISC-V 用的是小端序。

2.5 Representing Instructions in the Computer

之前我们学的 ld，x5 什么的其实都不算真正的机器语言，真正的机器语言是它们的二进制表示形式。

一长串二进制数，看一眼眼睛都疼，我们一般采用十六进制来书写这些二进制数，你可以 以 0x 开头来指明这是一个十六进制数。

怎么学这个机器语言，或者说学到什么程度呢？自然不会让你一条条去背，事实上，类型相同的指令具有类似的格式，而我们只需记住这些格式即可。

但也没那么简单，不同的指令类型有不同的指令格式，具体来说，可以分为以下六类：

  • R-type：inst rd,rs1,rs2

    包括各种算数逻辑运算，比如加减、异或、位移等等

  • I-type：inst rd,rs1,imm 或 inst rd,imm(rs1)

    包括有常数参与的算术逻辑运算，load 操作，jalr 指令，其中 imm 只有 12 位

  • S-type：inst rs2,imm(rs1)

    包括 store 操作

  • SB-type：inst rs1,rs2,imm[12:1]

    包括条件分支操作，imm[0] 默认 0，故不存储

  • UJ-type：inst rd,imm[20:1]

    包括无条件分支操作，imm[0] 默认 0，故不存储

  • U-type：inst rd,imm[31:12]

    高位立即数相关操作

你可能有点糊涂，为什么要分类呢？虽然有些指令还没提到，但可以看出，相同类型指令的书写也相同（除了 I-type ）。但分类的主要依据其实和指令的二进制形式有关。先放一张好图（这张图几乎把我想讲的讲完了）：

2.8 Supporting Procedures in Computer Hardware

这一节主要在讲 Procedure，中文直译为“过程”，感觉翻译成“函数”也没什么关系，毕竟这玩意和 C语言中的函数还是太像了

当然，这一节还提到寄存器、栈之类的东西，可能是这章信息量最大的一节

Procedure

Procedure 具有很多与 C 语言中函数类似的特征，比如可复用，能使程序易于理解，可以传递参数，可以有返回值

一个 Procedure 的执行过程可以概括为下面六步：

  1. 将所需参数放在指定的位置（一般是 x10-x17 这 8 个寄存器）
  2. 转交程序控制权（通过 jal 指令改变 PC）
  3. 获取所需的存储资源（调用内存？不太清楚）
  4. 执行目标任务
  5. 将结果放在指定位置（一般是复用 x10-x17 这 8 个寄存器）
  6. 交还控制权（通过 jalr 指令改变 PC）

jalr 应该是 jump and link 再加上 return 的意思。0 是偏移量，jump 即跳转到 x1+0 代表的指令地址位置，也就是函数的原调用位置 ，link 本来是写入当前指令地址，但由于写入的寄存器是 x0 ，所以可以认为 link 这步无效（有意为之，因为我们一般也不需要保存函数返回时的地址）

总之，每次调用都会产生一对 jal 和 jalr，前者负责调用，后者负责退出，二者存在配合。

Register

特定序号的寄存器在程序中承担特定功能，像之前提到的 x10-x17，专门负责为 Procedure 的调用存储参数和返回值。上面指令格式中的 x1 也不是瞎写的，它是专门用来存储返回地址的寄存器。还有一类寄存器，常常在 Procedure 中作为临时的局部变量使用，分别为 x5-x7 和 x28-x31。这些寄存器都与 Procedure 的调用有关。

Stack

我们知道，在 C 语言中，函数中的局部变量一般存储在栈中，函数结束后这些局部变量就被丢弃了。当 Procedure 中用到的局部变量太多，仅凭上面的 temp reg 和 x10-x17 应付不过来时，我们也会讲溢出的数据暂时存入栈中。

栈是内存的一部分，不直接存入内存中应该是因为用栈维护更方便管理内存，而且栈这个数据结构在函数调用，尤其是涉及函数嵌套过程中有天然优势（我记得算法竞赛中，为了加速或者防止函数递归太深导致栈溢出，会将函数调用通过栈模拟的方式实现）

栈指针之前提过，由 x2 负责，有时也叫 sp ，即 stack point 之意。由于历史原因，栈是从高地址开始往下存的，也就是你想往栈里 push 一个值，你得将 sp 减去这个值的 size 而不是加。

栈调用一般会和函数嵌套关联（ Procedure 和函数差不多，懒得切输入法了），比如一个函数在调用自己时，由于为了防止新值覆盖旧值，我们通常会把参数，中间的运算结果以及返回地址通过栈先封存起来，待函数递归完成再取出来

具体例子可以去看一个斐波那契函数的实现，当然写法不止一种，不要太死板了
