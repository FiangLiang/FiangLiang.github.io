Dyanmic Programming

Isshiki修’s Notebook 中没有提及这一部分的笔记，所以只参考了讲义和 Mooc

Mooc 上似乎是举了几个例子，嗯，DP 确实是结合例子才更好理解

要有心理准备吧，我记得当时一个01背包问题就卡了我好久，不过理解以后感觉其他 DP 也差不多就那样了

对的，DP 分为几种，最简单的是线性 DP，一般用一维数组实现

线性代数还在追我

矩阵乘法优化问题，假设有 nnn 个矩阵作乘法，已知第 kkk 个矩阵大小为 ak×bka_k\times b_kak​×bk​，且满足 bk=ak+1b_k=a_{k+1}bk​=ak+1​，求所需的最小乘法次数

我们知道一个 a×ba\times ba×b 和一个 b×cb\times cb×c 的矩阵相乘是一个三重循环，共需要进行 a×b×ca\times b\times ca×b×c 次乘法，而矩阵乘法是服从乘法结合律的，所以我们改变乘法的顺序，所需的总乘法次数可能有所不同

第一眼看上去可能有点棘手，不太可能会向 DP 去考虑（如果没学过 DP 的话）。如果是暴力法，枚举每次相乘的是哪两个相邻的矩阵，复杂度为 O((n−1)!)O((n-1)!)O((n−1)!)，这是我们不能接受的

那么怎么 DP 呢？之前说过，主要难点在于找到这个递推式，这么找呢？我们可以先尝试想想如何“记忆化搜索”。

Floyd Shortest Path Algorithm

一张图，有 nnn 个节点，求任意两点之间的最短路

Floyd 其实不是一个高效的最短路算法，复杂度为 O(n3)O(n^3)O(n3)，用 nnn 次 Dijkstra 也能得到相同的效果，复杂度（无负边权）仅为 O(n2log⁡n)O(n^2 \log n)O(n2logn)

但 Floyd 代码短，此乃一胜；Floyd 写起来快，此乃二胜；Floyd 写起来不用脑子，此乃三胜

综上，Floyd 完爆 Dijkstra

这样看上去也像一个 DP 了，但为什么 dp 数组有三维？这个递推式又是什么意思呢？

dp[i][j][k] 第一二维表示这是节点 iii 到节点 jjj 的最短路径，第三维 kkk 表示的是最短路只能经过前 kkk 个结点。这很关键，当我们难以发现递推关系时，不妨像这样为 dp 加上一个维度，这可能有利于我们得出递推式

递推式右边有两部分，如果 kkk 用不上就继承 dp[i][j][k-1] ，用的上就更新，具体可以看讲义，理解有点困难是正常现象

Floyd 还可以用来判断图的联通性，其实就是我们学过的 Warshall 算法（计算传递闭包）

Floyd 还可以用来判断是否存在负环，只需检测是否存在 dis[i][j]<0 即可，虽然不如 Bellman-Ford 算法那样高效

Product Assembly

最后一个问题比较简单，也比较经典，是一个线性 DP

先总结一下解决 DP 问题的步骤：

  1. 分析组成最优解的最优子结构
  2. 写出递推式
  3. 按一定顺序完成递推
