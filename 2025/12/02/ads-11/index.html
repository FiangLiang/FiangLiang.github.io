Approximation

参考了 ADS 的讲义、Mooc 以及 Isshiki修’s Notebook

基本概念

三个期望

一般的，对于一个最优化问题的算法，我们对其有三个期望：

  1. 算法能找到最优解
  2. 算法能高效运行
  3. 算法是通用的，能解决所有情况下的问题

对于 NP 完全问题，我们目前无法同时做到以上三点，因此需要做出一点妥协

舍弃第二个期望——我们采用指数级的算法去解决 NP 完全问题，对于小规模的数据可以接受

舍弃第三个期望——我们可以找到一些任意解决的特例

舍弃第一个期望——我们只需保证找到的解与最优解“相差不大”，我们称这种算法叫做近似算法

近似比

如何衡量这种**“相差不大”呢？我们引入近似比**的概念。

假设对于某类问题，其中一个实例记为 III，我们设计了近似算法 AAA，定义：

  • A(I)A(I)A(I) 为算法 AAA 在实例 III 上得到的解
  • OPT(I)OPT(I)OPT(I) 为实例 III 的最优解

我们定义近似比 ρ\rhoρ 为：

ρ=max⁡{OPT(I)A(I),A(I)OPT(I)}\rho=\max\{\frac{OPT(I)}{A(I)},\frac{A(I)}{OPT(I)}\} ρ=max{A(I)OPT(I)​,OPT(I)A(I)​}

我们称 AAA 为该问题的 ρ−\rho-ρ−近似算法，

至于为什么要取 max⁡\maxmax，因为该问题可能为极大化问题，也可能为极小化问题，我们始终保证 ρ>1\rho>1ρ>1

近似算法复杂度

对于一个近似算法，我们不仅希望它的近似比较小，同时也希望它的复杂度是一个较小的多项式

下面我们将介绍近似算法中几类常见的复杂度，我们设 ε=ρ−1>0\varepsilon=\rho-1>0ε=ρ−1>0，设实例 III 的规模为 ∣I∣|I|∣I∣

  1. PTAS（多项式时间近似方案）

    如题，范围很广，只需复杂度为多项式即可，由于复杂度一般和 1ε\frac{1}{\varepsilon}ε1​ 有关，一般将 PTAS 的复杂度记为

    T=O(∣I∣f(1ε))T=O(|I|^{f(\frac{1}{\varepsilon})}) T=O(∣I∣f(ε1​))

    由于 ε\varepsilonε 一般很小，虽然都是多项式，类似 O(∣I∣1/ε)O(|I|^{1/\varepsilon})O(∣I∣1/ε) 这样的复杂度就会比较糟糕

  2. EPTAS（Efficient PTAS）

    PTAS 的子集，在 FTAS 基础上，它还要求算法 AAA 的复杂度为 O(∣I∣c)O(|I|^c)O(∣I∣c)，其中 ccc 是与 ε\varepsilonε 无关的常数也可以将其的复杂度记为

    T=O(∣I∣O(1)f(1/ε))T=O(|I|^{O(1)}f(1/\varepsilon)) T=O(∣I∣O(1)f(1/ε))

  3. FPTAS（Full PTAS）

    EPTAS 的子集，在 EPTAS 的基础上，要求算法 AAA 的复杂度关于 ∣I∣|I|∣I∣ 和 ε\varepsilonε 都呈多项式，可以将它的复杂度记为

    T=O(∣I∣O(1)(1/ε)O(1))T=O(|I|^{O(1)}(1/\varepsilon)^{O(1)}) T=O(∣I∣O(1)(1/ε)O(1))

装箱问题

给定 nnn 个不同尺寸的物品，要求将所有物品放入容量给定的箱子中，求最少需要使用的箱子数量

这是一个 NP-完全问题，证明参见讲义

下面我将介绍几种不同的近似算法

Next Fit ( NF )

打开一个箱子，放入第一个物品，后续物品到达时，如果箱子有足够空间，就直接放入，否则关闭该箱子（以后不再访问这个箱子），再打开一个新的箱子，将该物品放进去

这个算法非常简单，你可以用线性复杂度实现这个算法。但我们同样可以看到，这种算法也产生了很多浪费——有些箱子还有空间却被提前关闭了

所以近似效果如何呢？可以证明，对于任意实例 III，满足

NF(I)≤2OPT(I)−1NF(I)\le2OPT(I)-1 NF(I)≤2OPT(I)−1

渐进近似比 α=2\alpha=2α=2（渐进比就是当 OPT(I)OPT(I)OPT(I) 充分大时，可以忽略常数的影响），考虑到现在还找不到近似比小于 3/23/23/2 的算法，感觉也能接受

下面将介绍更强的算法

Any Fit

这是一类 Fit 方案，只有当前面的箱子都装不下时，才打开新的箱子

Any Fit 根据填入箱子的优先级可以分为以下三种类型：

  • First Fit ( FF )：选择最早打开的箱子优先填入
  • Best Fit ( BF )：选择剩余空间最小的箱子优先填入
  • Worst Fit ( WF )：选择剩余空间最大的箱子优先填入

这三个算法都是典型贪心算法，每一种的近似效果都不会比 NF 差

直观看去似乎 BF 的贪心策略最为聪明，但其实 FF 和 BF 的渐进近似比是一样的，都是 1.7，；WF 看起来是最差的算法，事实上确实如此，WF 的渐进近似比与 NF 一致，均为 2.0

First Fit Decreasing ( FFD )

之前我们讲述的算法都是在线算法，都是根据输入一步步决策的，而我们知道，离线算法相比在线算法，有更多的操作空间，所以一般会比在线算法更加高效

FDD 就是一种离线算法，它的思路很简单：预先将所有的物品按照尺寸大小排序，再使用 First Fit

该算法的渐进近似比为 1.5，为目前已知的最优渐进比

各种算法渐进比的证明可以参见讲义，因为比较复杂，所以懒得搬了

0-1 背包问题

在动态规划一节中，我们知道（虽然我没讲），用动态规划解决 0-1 背包问题复杂度为 O(nC)O(nC)O(nC)

但这里的 O(nC)O(nC)O(nC) 是一个伪多项式时间，因为“输入规模”指的是描述输入的 bit 数，假设背包容量 C 在输入中占 mmm 位，复杂度就为 O(n2m)O(n2^m)O(n2m) ，也就是指数规模（详细原因参见讲义 NP 问题一节）

所以说出来你可能不信，但 0-1 背包是 NP 问题，而且是 NP 完全问题，下面我们尝试用近似算法解决这个 0-1 背包问题

贪心算法

下面介绍一个近似比为 2 的近似算法

先提一个 0-1 背包的简单版本——分数背包，也就是你可以把一个物体进行切分，切分的部分按照切分比例换算器价值和重量

分数背包可以采用一个很显然的贪心算法解决，只需将物体按照 “价重比” 排序，每次选择价重比最大的物体，无法选择完整物体时，就切下该物体的一部分将背包装满

下面我们介绍这个 2-近似算法：

  1. 类似分数背包，将物体按照价重比排序，每次优先选择价重比更大的物体，得到一个解
  2. 再将物体直接按照价值排序，每次优先选择价值更大的物体，得到一个解
  3. 取两个解的较大值作为算法的解 PgreedyP_{greedy}Pgreedy​

下面我们简单证明一下 近似比为 2 的结论

设对于某一实例 III，分数背包最优解为 PfracP_{frac}Pfrac​，0-1 背包最优解为 POPTP_{OPT}POPT​，所有装得下的物体中最大价值为 pmaxp_{max}pmax​

我们有如下不等式：

pmax≤Pgreedy≤POPT≤Pfracp_{max}\le P_{greedy}\le P_{OPT}\le P_{frac} pmax​≤Pgreedy​≤POPT​≤Pfrac​

从左到右理解性地推理一下就行，然后是下面这串不等式：

POPTPgreedy≤PfracPgreedy≤Pgreedy+pmaxPgreedy≤1+pmaxPgreedy≤2\frac{P_{OPT}}{P_{greedy}}\le \frac{P_{frac}}{P_{greedy}}\le \frac{P_{greedy}+p_{max}}{P_{greedy}}\le1+\frac{p_{max}}{P_{greedy}}\le2 Pgreedy​POPT​​≤Pgreedy​Pfrac​​≤Pgreedy​Pgreedy​+pmax​​≤1+Pgreedy​pmax​​≤2

为什么 Pfrac≤Pgreedy+pmaxP_{frac}\le P_{greedy}+p_{max}Pfrac​≤Pgreedy​+pmax​ 呢？因为我们可以把分数背包问题最后不完整的部分舍去，而舍去的部分不可能大于最大价值 pmaxp_{max}pmax​

这样我们就知道近似比 ρ≤2\rho\le 2ρ≤2，下面我们需要证明 ρ≥2\rho \ge 2ρ≥2，讲义中构造的例子似乎有问题，我一时半会也想不到简单的例子，就略过这一步了

动态规划算法

这一节开头提到的 DP 算法，其中 dp[i][j] 表示将前 iii 个物品放进容量为 jjj 的背包中能得到的最大价值，但我们知道这个算法的 O(nC)O(nC)O(nC) 其实是一个伪多项式复杂度，下面我们采用一种新的 DP 思路

其实就是改了一下 dp 的第二维含义，将背包容量改为了背包价值，dp[i][j] 表示将前 iii 个物品放入总价值为 jjj 的背包中能得到的最小重量，递推式类似，复杂度为 O(nV)O(nV)O(nV)，其中 VVV 为所有物品总价值。设 vmaxv_{max}vmax​ 为所有物品中的最高价值，有 nvmax≤Vnv_{max}\le Vnvmax​≤V，所以复杂度也可写作 O(n2vmax)O(n^2v_{max})O(n2vmax​)

但由于 vmaxv_{max}vmax​ 和 nnn 没关系，所以这也是个伪多项式复杂度。为了得到一个多项式算法，我们需要建立 vmaxv_{max}vmax​ 和数据规模 nnn 之间的关系——我们可以对输入的价值进行等比例的缩小，使 vmaxv_{max}vmax​ 能缩小到 nnn 的多项式。

假设缩小比例为 bbb，由于动态规划需要使下标为整数，所以令缩小后的物品价值 vi′=⌈vi/b⌉v_i'=\lceil v_i/b \rceilvi′​=⌈vi​/b⌉，也就是上取整（下取整也行），最后我们只需要将结果放大 bbb 倍即可，中间会损失一些精度，所以这是一个近似算法。

我们可以取缩小比例

b=εvmaxnb=\frac{\varepsilon v_{max}}{n} b=nεvmax​​

其中 ε\varepsilonε 为我们想要的近似比率（理论上 ε\varepsilonε 可以无限小），所以原复杂度可以写作 O(n3/ε)O(n^3/\varepsilon)O(n3/ε)，这其实满足 FPTAS 的形式

接下来我们需要证明 ε\varepsilonε 就是我们想要的近似比率，这部分比较复杂，有兴趣的可以去讲义看看

k - 中心问题

k-center problem：输入 nnn 个点位置以及一个整数 kkk，要求选择 kkk 个点作为中心，使得每个点到最近中心的距离的最大值 r(C)r(C)r(C) 最小化

最近事比较多，这里简单讲一下思路：

  • 我们发现，如果能找到这个 r(C)r(C)r(C) 的最小值，我们可以用 O(n2)O(n^2)O(n2) 时间找到符合的 kkk 个中心
  • 为了找到 r(C)r(C)r(C) 最小值，我们采用二分法，将其变为判定问题，即，给定 r(C)=rr(C)=rr(C)=r，是否存在 kkk 个以点为圆心，rrr 为半径的圆，使其能覆盖图中所有点
  • 判定函数是一个用贪心实现的近似算法，它只能用来判定 rrr 能否作为一个上界，具体实现如下：
    • 设 CxC_xCx​ 为选择的中心点，初始化为 ∅\emptyset∅，SxS_xSx​ 为为被覆盖的点，初始化为输入的点集
    • 当 Sx≠∅S_x \ne \emptysetSx​=∅ 时，重复如下操作：
      • 随机选择 si∈Sxs_i\in S_xsi​∈Sx​，将其插入 CxC_xCx​ 中，将其从 SxS_xSx​ 中删除（选取任意未被覆盖点作为中心点，其一定被覆盖）
      • 删除 SxS_xSx​ 中能被以 sis_isi​ 为圆心覆盖的点（删除新的被覆盖点）
    • 所有点被覆盖后，若 ∣Cx∣≤k|C_x|\le k∣Cx​∣≤k，返回 yes，否则返回 no
  • 近似比为 2。我们假设最优解为 rxr_xrx​，可以推导得到 2rx2r_x2rx​ 是能通过上面的判定函数的。因为最优解对应的某个中心圆里任意两点距离不超过 2rx2r_x2rx​，所以以其中任意一点为圆心，以 2rx2r_x2rx​ 为半径的圆都能覆盖圆内其他的点。所以无论随机选择了那些点， 2rx2r_x2rx​ 都是可行的
