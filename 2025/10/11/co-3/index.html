Addition, subtraction and ALU

数逻课上讲过的内容会在下面略过

Addition & substraction

二进制加法的实现很简单，减法也只需将减数取补码后相加

关于 Overflow 的问题，一般只考虑有符号数，一共四种可能情况，可根据结果符号位与其前一位的异或结果直接判断

硬件层面能检测溢出情况，然后将当前指令的地址存储到一个特别的寄存器 EPC 中，然后跳转到特定例程：纠正程序、返回错误代码、或者中止程序

Speed considerations

上面加法使用的是行波加法器，即 Ripple Carry Adder，是由 1 位加法器串联形成的，每一个 1 位加法器都要等待前面的完成计算才能开始计算，这种实现方式会很慢。

为了加速，我们可以搭建额外的电路来提前求出所需的进位

Booth’s Algorithm

一种很新颖的乘法，和我们之前提到的“龟速乘”不同，它在做乘法时会先对乘数进行处理。

为什么要处理呢？有的时候，乘数中可能会出现一串连续的 1 ，按之前的方案，每个 1 都会造成一次加法，Booth’s Algorithm 根据这一点，将一串连续的 1 分解成了两个 1，一个做加法，一个做减法

Division

也介绍两个版本

V1

其实除法和乘法的实现类似，只不过乘法是由低位到高位不断累加，而除法是从高位到低位不断去试着做减法

除数寄存器有 128 位，一开始除数在寄存器的最右边，相当于左移了 64 位，被除数在余数寄存器中。每次都用余数减去除数，得到负数就加回去，得到正数就将商寄存器的当前位置 1，然后除数寄存器和商寄存器进行位移，在进行下一次操作。

V3

为什么这里是 V3？因为这里的改进几乎是照抄了乘法器的 V3 版本。

将除法寄存器和 ALU 都降低到了 64 位，去掉了商寄存器，将商的结果保存到余数寄存器的右边

Signed Divsion

不能像乘法一样用 Booth’s Algorithm “逃课”了，只能老实把负数转化为无符号数进行计算，最后再判断商的符号位，余数则需根据被除数的正负来确定符号。

Fast Divsion

除法不能用并行计算，有一种叫做 SRT Division 的方法，通过预测来提高除法速度，类似与前面提到过的选择进位加法器，不过应该不太重要

Floating point numbers

之前我们讨论的都是整数范围的运算，为了将小数引入计算，我们需要用到浮点数

浮点数是怎样保存一个小数的呢？很简单，一部分用来保存有效位数，一部分用来保存小数点的位置，留下一位来作为符号位即可。下面介绍一套比较通用的浮点数表示法。

IEEE 754 standard

在这套体系中，浮点数分为 32 位的单精度浮点数和 64 位的双精度浮点数，分别对应 C 语言中的 float 和 double。浮点数由三部分组成：符号 SSS，指数 EEE ，尾数 FFF。指数影响浮点数的范围，尾数影响浮点数的精度。对一个浮点数，可以将其表示为：

(−1)S×F×2E(-1)^S\times F\times 2^E (−1)S×F×2E

呃，上面的表达式仅用于理解，事实上，IEEE 754 standard 在此基础上做了些许改进

然后是位数的分配：

可以看出，对于单精度，有 8 位分配给了指数，23 位分配给了尾数；对于双精度，有 11 位分配给了指数，52 位分配给了尾数。

你可能会问：啊，为什么 64 位的双精度浮点数指数只有 11 位？这样能表示的范围不会很小吗？其实不小，事实上，在 C 语言中，float 能表示的范围已经比 long long 还大了（指数表示法是这样的）。

回归正题，之前提到 IEEE 754 standard 与传统的二进制浮点数表示方法不同，哪不同呢？

首先，除了 0，可以发现 FFF 都是 1.xxxx1.xxxx1.xxxx 的形式，所以 IEEE 754 中干脆就不存这个 1 了，尾数部分都用来存 .xxxx.xxxx.xxxx 的部分。

其次，由于 EEE 是个有符号数，IEEE 754 不希望看到有符号数（我也不知道为啥），所以引入了一个偏移 biasbiasbias ，也就是说，我们保存的不是 EEE，而是 E+biasE+biasE+bias，对于单精度，biasbiasbias 为 127，对于双精度，biasbiasbias 为 1023。

综上，用 IEEE 754 standard 表示的一个数，其值为：

(−1)S⋅(1+fraction)⋅2exponent−bias(-1)^S\cdot (1+fraction)\cdot 2^{exponent-bias} (−1)S⋅(1+fraction)⋅2exponent−bias

Floating point addition

众所周知：

浮点数怎么相加？没有你想的那么复杂！一共六步：

  1. 指数对齐，小数向大数对齐
  2. 对齐后的小数部分相加
  3. 规格化，也就是将结果位移直至符合 IEEE 754 standard
  4. 判断是否溢出（指数过大或过小）
  5. 舍入
  6. 舍入可能产生进位，需再次规格化

Floating point multiplication

第一步是指数相加，第二步是有效数字相乘，最后多了一步判断符号位，其他和浮点数的加法相同

Accurate Arithmetic

浮点数运算时会额外添加两位——保护位（guard）和舍入位（round），用于在舍入时减少精度损失，有时也会在舍入位右边增加粘滞位（sticky）
