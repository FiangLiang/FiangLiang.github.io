在关系数据模型中，数据被组织为表格的形式。表格是由行和列组成的，其中每行代表一个实体，每列代表一个属性。表格中的每个单元格包含一个值。

1 关系

先来解释几个概念

  • 关系（relations）
    • 数学定义：对于给定的集合 D1,D2,...,Dn(Di=aij∣j=1...k)D_1,D_2,...,D_n(D_i=a_{ij|j=1...k})D1​,D2​,...,Dn​(Di​=aij∣j=1...k​)，关系 rrr 为笛卡尔积 D1×D2×...×DnD_1\times D_2\times ...\times D_nD1​×D2​×...×Dn​ 的子集
    • 元组（tuple）是关系里的一组属性值，关系可以理解为是由 n 元组（a1j,a2j,...,anja_{1j},a_{2j},...,a_{nj}a1j​,a2j​,...,anj​）构成的集合，其中 aij∈Dia_{ij}\in D_{i}aij​∈Di​
    • 属性（attribute）
      • 每个属性都有一个名称
      • 每个属性允许值的集合称为该属性的域（domain）
      • 属性值一般要求具备原子性（atomic），即不可分割（indivisible）
      • 特殊值 null 是所有域的成员
    • 关系是无序的，同一个关系内，不能存在重复的元组
  • 关系模式（relation schema）
    • 描述关系的结构，包含了属性值及其对应的域
    • 假如有属性 A1,A2,...,AnA_1,A_2,...,A_nA1​,A2​,...,An​，那么 R=(A1,A2,...,An)R=(A_1,A_2,...,A_n)R=(A1​,A2​,...,An​) 就是一个关系模式，而 r(R)r(R)r(R) 就是在关系模式 RRR 上的一个关系
  • 关系实例（relation instance）
    • 在某个时刻上，关系中数据的快照（snapshot）

2 Key

在关系模型中，一个键（Key）是一个属性或属性集合，它可以唯一标识一个元组。

Key 可以分为以下几类：

  • Superkey（超键）

    一个超键是一个属性集合，它可以唯一标识一个元组

  • Cadidate Key（候选键）

    最小数量的超键，也就是说候选键的子集无法构成超键

  • Primary Key（主键）

    一种候选键，且由数据库设计者显式选择的键（通常用下划线标出）

  • Foreign Key（外键）

    • 外键就是一张表里某一列属性，它的值都可以在另一张表的主键中找到
    • 外键需要参照完整性约束，即外键的值必须在被参照的表的主键中存在

4 关系查询语言

关系查询语言是用于查询数据库的语言，它可以分为两类：

  • 关系代数

    关系代数是一种形式化的查询语言，它使用一组操作来操作关系。这些操作包括选择、投影、连接、并、差、交等。

  • 关系演算

    关系演算是一种声明性的查询语言，它描述了所需的数据，而不是如何获取数据。关系演算分为两种：元组关系演算和域关系演算。

上面两类语言在计算上是等价的，可以相互转换。我们一般使用关系代数来查询数据库。

4.1 六类基本操作

Select

选择操作，选择满足条件的元组

其中 ppp 称为选择谓词（selection predicate），可通过连接符（∧,∨,¬\land,\lor,\lnot∧,∨,¬）连接多个谓词

Project

投影操作，选择满足条件的属性，记作 ∏A1,A2,...,Ak(r)\prod_{A_1,A_2,...,A_k}(r)∏A1​,A2​,...,Ak​​(r)，其中 A1,A2,...,AkA_1,A_2,...,A_kA1​,A2​,...,Ak​ 是属性名称

运算结果为包含指定 kkk 列的关系，没有被指定的列会被排除在外，投影结果中的重复行会被自动删除

Cartesian Product

笛卡尔积操作，得到一个集合，里面包含了两个输入关系可能的所有元组组合

若 r,sr,sr,s 分别有 n1,n2n_1,n_2n1​,n2​ 的元组，那么它们的笛卡尔积有 n1⋅n2n_1\cdot n_2n1​⋅n2​ 个元组

Union

并操作，合并两个关系

被合并的两个关系必须满足：两个关系的属性数目相同，且属性的域是兼容的

Set Difference

差操作，返回两个关系的差集

得到一个关系，其包含了出现在第一个输入关系，但不出现在第二个输入关系的元组

4.2 拓展操作

Intersection

交操作，返回两个关系的交集

可用差操作定义交操作：r∩s=r−(r−s)r\cap s=r-(r-s)r∩s=r−(r−s)

Natural Join

自然连接操作，返回两个关系的连接，记作 $ r\Join s$

自然连接是一种特殊的笛卡尔积，它只返回那些在连接属性（R∩SR\cap SR∩S）上相等的元组。

Theta Join

Theta 连接，又称条件连接，记作 r⋈θsr\Join_\theta sr⋈θ​s

也就是在自然连接的基础上加上一个条件

Outer Join

外连接扩展了传统连接的语义，允许我们保留那些在连接条件上不匹配的元组，从而在查询中获取更完整的信息

外连接分为三种：

  • 左外连接（left outer join）⟕
    • 先执行一个普通的连接
    • 保留左边关系中所有元组（即使在右边没找到匹配元组）
    • 对于那些没有匹配元组的行，右边关系对应的属性值用 null 填充
  • 右外连接（right outer join）⟖
    • 先执行一个普通的连接
    • 然后保留右边关系中所有元组（即使没在左边关系中找到匹配元组）
    • 对于那些没有匹配元组的行，左边关系对应的属性值将用 null 填充
  • 全外连接（full outer join）⟗
    • 左外连接和右外连接结果的并集
