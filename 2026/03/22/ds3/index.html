Lec 3: Introduction to SQL

第一章说到，一门数据库系统的语言系统主要由数据定义语言（DDL）和数据操纵语言（DML）组成，我们先从相对简单的 DDL 讲起

1 SQL Data Definition

DDL 主要负责指定每个关系的模式、每个属性的值域以及完整性约束

1.1 Basic Types

SQL 标准主要包含以下几种数据类型：

  • char(n)：定长字符串，由用户指定长度 n
  • varchar(n)：不定长字符串，由用户指定最大长度 n，定长和不定长字符串的比较可能会出现问题
  • int：整数
  • smallint：较小的整数
  • numeric(p, d)：定点 (fixed point) 数，由用户指定有效位数 p 以及小数点后的位数 d
  • real / double precision：分别对应单精度浮点数和双精度浮点数，其精度依赖于机器
  • float(n)：浮点数，由用户指定最低精度位数 n

SQL 标准还定义了以下几种内置类型：

  • date：日期,包括四位数年份、两位数月份和两位数日期，如 ‘1999-12-31’
  • time：时间,包括小时、分钟和秒，如 ‘12:30:00’
  • timestamp：时间戳,包括日期和时间，如 ‘1999-12-31 12:30:00’
  • interval：时间间隔
    • 从一个日期/时间/时间戳值中减去另一个得到时间间隔值（有点像指针），时间间隔值可以添加到日期/时间/时间戳值

SQL 中有许多函数用于处理各种类型的数据及其类型转换，但各数据库系统中函数的标准化程度不高

比如下面两类：

  • SQL Server：char(65), substring(s, start, length), getdate(), datalength(‘abc’),
  • Oracle：chr(65), substr(s, start, length), sysdate, length(‘abc’)

2.5 String Operations

关于字符串操作，有以下要点：

  • SQL 的字符串要用单引号 ’ 包裹

  • 字符串比较在大部分数据库系统中是大小写敏感的

  • SQL 支持以下字符串函数：

    • 拼接 (concatenation)：运算符为 ||
    • 获取字符串长度
    • 提取子字符串
    • 将字符串转化为大写 / 小写（分别对应 UPPER(s) 和 LOWER(s)）
    • 移除字符串末尾的空格：TRIM(s)

  • SQL 支持模式匹配(pattern matching)（或者叫模糊匹配，类似正则匹配），用到运算符 LIKE

    LIKE 支持的字符匹配方式：

    • %：匹配任意字符串
    • _：匹配任意单个字符
    • [abc]：匹配a、b或c
    • [a-z]：匹配a到z之间的任意字符
    • [^abc]：匹配不是a、b或c的任意字符

2.8 Duplicates

虽然在传统的关系理论中，不会出现重复数据，但是在实践中，有时我们需要重复的数据。

SQL 的查询语句默认支持多重集（multiset）的运算符。如果不希望出现重复记录，则添加 DISTINCT 关键字

3 Set Operations

SQL 中包含交、并、差等集合操作，分别用 UNION, INTERSECT, EXCEPT 表示

使用这些运算符后会自动消除重复记录，如果想要保留重复记录，需要在集合运算关键字后加上 ALL 关键字，假如有一个元组，在关系 r, s 内分别出现了 m, n 次，那么该元组在

  • 关系 r UNION ALL s 中出现 m + n 次
  • 关系 r INTERSECT ALL s 中出现 min(m, n) 次
  • 关系 r EXCPET ALL s 中出现 max(0, m - n) 次

4 Null Values

元组的某些属性可能是空值，记作 null，表示未知值或不存在的值

有以下知识点：

  • 任何包含 null 的算术表达式的结果为 null
  • 任何包含 null 的比较结果为 unknown（除了 IS NULL 和 IS NOT NULL）
  • SQL 的逻辑表达式的结果有 3 种：true、unknown、false

至此，我们已经学习了大部分的查询语句，它们的执行顺序如下：

  1. FROM：确定数据来源
  2. WHERE：根据条件过滤原始表中的行
  3. GROUP BY：将数据按指定列进行分组
  4. HAVING：根据条件过滤分组后的结果
  5. SELECT：选择最终要返回的列
  6. ORDER BY：对结果进行排序
  7. LIMIT：限制返回的行数
